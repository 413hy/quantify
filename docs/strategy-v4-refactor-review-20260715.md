## V3 与目标需求的错位

1. V3 观察 SOL、BNB、XRP、DOGE、ADA；V4 用 BTC、ETH 替换实际结果为负且执行质量更差的
   DOGE、ADA，同时保留 BNB、SOL、XRP 以增加经过确认的机会来源。
2. V3 用 1m/5m PA、5 秒成交窗口和 10 秒评估轮次生成 35–60 bps 目标；参考记录的目标中位数
   约 21 bps，最快 24 秒平仓，需要更高频的因果订单簿特征。
3. V3 的 `signal_quality_score` 是未校准的加法分数。实际盈利和亏损样本的高分重叠，分数并不
   是“先到止盈而非止损”的概率。
4. 当前入场和保护退出均按市价/taker 成本估计。20 bps 毛目标扣除约 8 bps 双边手续费和
   2 bps 常规不利滑点，只剩约 10 bps；100 USDT 名义价值才约等于 0.10 USDT 净目标。
5. V3 固定不加仓、单笔最大净亏损 0.35 USDT，这一点是正确的风险边界，但意味着不能借用
   倍增补仓制造出来的已平仓胜率。

## V4 应保留与拒绝的部分

可以借鉴：

- BTCUSDT、ETHUSDT 优先，并保留 BNBUSDT、SOLUSDT、XRPUSDT 的固定五币池；
- 交易所允许时使用 100x，但以固定保证金和绝对亏损预算决定数量；
- 18–22 bps 的费用前短目标；
- 无持仓时间到期退出，仍由目标或结构失效退出；
- 有信号才交易，不为了补满并发槽位而开仓。

必须拒绝：

- 全仓模式下的无限或倍增加仓；
- “不平亏损仓所以已平仓胜率 100%”的统计口径；
- 只根据胜率选择模型，忽略平均盈亏、浮亏、保证金峰值和尾部风险；
- 将 40–150 USDT 峰值保证金交易描述成固定 1 USDT；
- 未经样本外验证就把 3/3 结果部署为真钱策略。

## 推荐的 V4 结构

1. **数据层**：为固定五币池维护 100–500ms 连续本地订单簿和逐笔聚合成交，生成
   1s、3s、10s 三个因果窗口；保留 1m/5m 只作为方向和波动环境，不直接充当短目标预测器。
2. **标签层**：为每个候选记录先触达 `+18/+20/+22 bps` 还是结构失效，并记录最大有利/不利
   变动、点差、真实费率和成交滑点。所有候选都产生 shadow 标签，无需真实下单才能扩大样本。
3. **信号层**：把加法质量分替换为按时间切分校准的目标先达概率；至少使用成交方向持续性、
   多窗口 CVD、microprice、盘口失衡变化、点差、短时波动、1m/5m 方向和距离局部结构的位置。
4. **准入层**：只有概率下界高于费用后保本概率才入场。若目标净赚 0.10 USDT、结构止损净亏
   0.35 USDT，理论保本胜率已是 `77.78%`，还需为模型误差预留余量。
5. **执行层**：研究 maker-first 的 `GTX` 限价入场和 reduce-only 限价止盈；短暂未成交即撤销，
   仅当 taker 后仍满足净边际时才追价。结构止损始终保留原生 stop-market，不能为了费率取消。
6. **仓位层**：每个币种最多一笔，保证金上限约 1 USDT，不逆势加仓；实际名义价值、费率、
   滑点和结构距离共同约束最大净亏损。
7. **验证层**：先收集至少 100 个样本外已完成标签，并按时间 walk-forward；报告手续费后
   Profit Factor、置信区间、最大连续亏损、MAE/MFE 和最大保证金，而不是只报告胜率。

Binance 官方资料也明确提示，高胜率配合小平均盈利和大平均亏损仍可能为负；copy trading
不能消除市场风险。USDⓈ-M API 的 `GTX` 表示无法成为挂单方就撤销，可用于研究 maker-first，
但不能保证成交。

## 已实现的 Testnet V4

当前实现先完成可由现有基础设施验证的部分：固定五币池；BTC/ETH/BNB/SOL/XRP 毛目标分别为
20/22/25/32/25 bps；点差最多 5 bps；主动成交额至少为近期中位数 2 倍；方向连续确认 3 轮；
保证金最多 1 USDT；预计费用后目标至少 0.10 USDT；预计结构止损净亏损最多 1 USDT；禁止
补仓；没有时间到期退出。生产仍保持 `RISK_LOCKED`。

100–500ms 连续本地订单簿和概率校准仍属于下一研究阶段。V4.3 已在 Testnet 部署有界
maker-first 入场：先尝试 `LIMIT + GTX`，未成交且追价不超过 3 bps 才使用市价兜底；这不是
生产准入，也不能描述成已经证明降低实际费用。
现有 Testnet `aggTrade` 流已对五个新交易池完成只读连通验证，但 BTC/ETH 尚无当前活动的历史
因果观测，因此 V4 初期结果仍必须视为新实验样本，不能沿用 V3 胜率。
