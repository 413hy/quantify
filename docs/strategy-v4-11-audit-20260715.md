# V4.11 需求符合性复审（2026-07-15）

## 结论

V4.11 的执行和路由符合当前明确需求：五币池中每个独立有效信号均可直接市价执行；活动仓位由
最新确认信号管理；同向可加仓、反向先平后换向；没有开仓持仓时间到期退出。Testnet 服务由
systemd 和确定性规则独立运行，`codex_dependency=false`。

策略仍是 `UNVALIDATED_TESTNET_EXPERIMENT`。切换前 V4.10 已建立 XRP、BTC、BNB 三个受原生
止盈止损保护的市价仓位；另一次 ETH 部分成交在实际费用复核失败后已安全平仓。尚无足够已平仓
样本证明胜率或收益，不能把“符合需求”解释为“已经看准市场”。

## 需求对照

| 需求 | 复审结果 |
| --- | --- |
| 所有币有信号就做 | BTC、ETH、BNB、SOL、XRP 全部参与趋势和宽度计划；每轮候选上限 5 |
| 不使用开仓限价 | 新仓、同向加仓和反向后的新仓只调用 `MARKET` |
| 预测要参与判断 | 前后 10 分钟综合预测绝对幅度不足 1 bps 时拒绝；同向/冲突分组留证 |
| 实时信号优先 | PA、宽度、订单流完成确认后，滞后线性预测不覆盖实时方向 |
| 非确认交易可不做 | 质量、活跃度、连续轮次、点差、动量和微观结构门控全部保留 |
| 最多五个活动币 | 不同币最多五个；不是必须补满 |
| 同向叠仓 | 仅新确认事件触发一次；整仓风险/费用允许才市价增加并重设保护 |
| 反向换向 | 撤旧保护、市价平旧仓，再市价执行最新确认方向 |
| 止盈止损退出 | 交易所原生 `STOP_MARKET`/`TAKE_PROFIT_MARKET`；没有时间到期平仓 |
| Codex 不在线仍运行 | 决策权威为 `TESTNET_DETERMINISTIC_RULE`，systemd 自动维持服务 |

## 当前信号覆盖证据

V4.10 最近五分钟曾生成 14 个计划，五币均有覆盖：ETH 7、XRP 3、SOL 2、BTC 1、BNB 1。
随后 XRP、BTC、BNB 均完成确认并建立仓位；其他计划主要停在成交活跃度或连续确认门控，而不是
被交易池排除。V4.11 把三候选上限提升为固定交易池大小，同一轮最多五个已确认计划都能进入执行器。

## 策略边界

- 快速宽度要求至少三币约 40 秒同向 2 bps，持续宽度要求至少三币约 110 秒同向 5 bps；
- 本币对应窗口动量低于门槛或超过 12 bps 时不追单；该限制可能错过已进入后段的行情，但用于
  避免事后追涨杀跌；
- 冲量计划要求主动成交同向，并由盘口或 microprice 至少一项确认；任一微观结构读数明显反向
  都会否决。趋势计划另要求至少一个 PA 周期同向且两个周期都不反向；
- 当前毛目标为 BTC 20、ETH 22、BNB 25、SOL 32、XRP 25 bps。市价实际费率和成交后保护会
  再次验证费用后至少 0.10 USDT；
- 成交后复核失败会以 `EXECUTION_FAIL_CLOSED` 安全平仓并完整记录手续费和净盈亏，不再只留下
  执行错误；
- 结构止损允许约 30–120 bps。以当前小目标与较宽止损计算，策略需要较高费用后胜率才能覆盖
  单次止损，必须用自然样本复核，不能声称百分百胜率。
