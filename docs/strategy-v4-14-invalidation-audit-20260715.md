# V4.13 漏单与止损失效审查

## 现场结论

服务每 10 秒正常完成五币观测，漏单和止损来自策略门控，不是进程、网络、密钥或 Binance API
故障。V4.13 在 212 轮内生成 8 个计划：1 个确认成交，6 个被线性预测判为反向，1 个被主动
成交倍率拒绝。

被倍率拒绝的是 2026-07-15T12:38:12Z XRP 多单。当时四币同涨、XRP 动量 7.66 bps、方向
预测 +5.87 bps，计划目标 1.1133765 后续触达。成交额倍率却为 0.0003；后两次 XRP 多头计划
的倍率又跳到约 1,622 与 37,206。该指标来自 Testnet 最近 5 秒聚合成交批次，不具备硬门槛所需
的稳定性。

反向开仓批准”。

## V4.14 修正

- 快速入口等待完整 12 轮上下文；
- Testnet 绝对成交额倍率改为诊断项，方向失衡和盘口确认继续生效；
- 三币反向宽度加本币反向动量直接触发 `SIGNAL_INVALIDATION`；
- 失效只平旧仓，不自动反手；新方向仍走完整入场门控；
- 不增加持仓时间到期退出。

这次修改针对可重放的确定性缺陷，不宣称未来胜率或收益保证。
