# ADR 0023：已确认信号的 Testnet 市价兜底 V4.10

- 状态：Accepted
- 日期：2026-07-15
- 范围：Binance USDⓈ-M Futures Testnet；不解锁生产交易

## 背景

V4.9 已删除额外 2–8 bps 回撤，做多加入最优买一、做空加入最优卖一，并保持 30 秒 GTX。
在一轮约 30–50 bps 的五币同步下跌中，策略生成 20 个计划，ETH、BNB、XRP 共 3 个计划通过
质量和活跃度确认并提交真实 Testnet 订单。3 张单都准确位于最优卖一、额外距离为 0 bps，
但全部在约 30 秒后以成交量 0 撤销。服务、凭据和 Binance API 均无错误。

因此本轮“零仓位”不是没有信号，也不是限价离盘口过远，而是 Testnet Maker 队列没有对手方
成交。继续延长等待会让超短线信号过时，并不能解决流动性问题。账户所有者随后明确授权：有效
交易信号出现时，必要情况下可以使用市价入场。

## 决策

1. 计划仍须通过现有 PA、订单流、成交活跃度、连续确认、费用后目标和风险门控；市价兜底不
   放宽任何信号标准。
2. 入场先在最优买一/卖一提交 `LIMIT + GTX`，实际墙钟最多等待约 3 秒，保留短暂 Maker 成交
   机会。完全未成交时撤销该单，随后使用相同数量提交 `MARKET`。
3. Maker 部分成交时不再提交市价余量，直接按实际部分成交数量复核并建立保护，避免超过计划
   数量。
4. 市价成交后使用真实成交均价重新量化止盈止损，并复核费用后目标至少 0.10 USDT、预计整仓
   净止损不超过 1.00 USDT；不满足时 fail-closed 平仓。
5. 同向加仓使用相同的 3 秒 Maker + 市价兜底，但挂单和兜底前都检查父仓仍然存在；旧仓先被
   止盈止损关闭时返回 `PARENT_POSITION_CLOSED`，不得重新开仓。
6. 执行证据使用 `MAKER_TIMEOUT_MARKET_FILLED` 或
   `MAKER_TIMEOUT_MARKET_PARTIALLY_FILLED` 区分市价兜底，不把它报告为 Maker 成交。

## 后果

市价兜底显著提高 Testnet 成交确定性，同时会支付 taker 手续费并承受盘口滑点。系统继续使用
约 1 USDT 保证金、交易所允许的最高初始杠杆和成交后风险复核来限制单次实验，但这些限制不能
保证盈利。应分别比较纯 Maker、Maker 后市价兜底的实际手续费、滑点、目标命中率和费用后结果。
