# ADR 0020：最新有效信号接管 Testnet 仓位 V4.7

- 状态：Accepted
- 日期：2026-07-15
- 范围：Binance USDⓈ-M Futures Testnet；不解锁生产交易

## 背景

V4.6 在 `active_trades` 中为每个币维护一个完整生命周期执行线程，从预测限价入场一直持有到
原生止盈止损平仓。策略确认层把 `active_trades` 中的币全部标记为 `ALREADY_IN_FLIGHT`，所以
仓位建立后产生的新信号无法到达执行器。同向信号不能增加仓位，反向信号也不能及时终止旧方向。

另起第二个执行线程会造成两个线程同时管理同一币种的仓位及 `closePosition` Algo 单，存在互相
撤单、重复平仓或留下未保护仓位的竞态，因此不能以并发下单方式实现仓位接管。

## 决策

1. 每个活动币种继续只有一个订单所有者，并增加线程安全的 `PositionSignalControl` 邮箱。策略层
   只把已经通过质量、活跃度及连续轮次门控的计划投递给该邮箱。
2. 等待预测限价成交的币仍然不可接管；只有真实成交且原生止盈止损已经确认的币，才继续参加
   活动仓位信号确认。
3. 同一方向、同一 setup、同一连续信号事件只投递一次。信号消失、被门控拒绝或方向变化后，
   再次确认才形成新事件，防止 10 秒评估循环无限重复加仓。
4. 同向信号继续使用方向化预测 `LIMIT + GTX`，最多等待约 30 秒，不使用市价追单。旧原生保护
   在加仓挂单期间保持有效；若旧仓先平，立即撤销加仓单。加仓成交后读取交易所整仓数量和加权
   开仓价，撤销旧保护并挂出新的整仓 `STOP_MARKET` 与 `TAKE_PROFIT_MARKET`。
5. 加仓数量除 1 USDT 单次保证金上限外，还使用旧仓止损、双边 taker 费和 12 bps 定仓滑点
   计算剩余风险。整仓预计净止损超过 1 USDT、费用后目标低于 0.10 USDT，或剩余风险不足交易所
   最小数量时，不提交加仓。
6. 最新有效信号与仓位反向时，执行器先撤销旧保护并平掉旧仓，结果记为 `SIGNAL_REVERSAL`。
   已确认的新方向交回 campaign，并绕过 60 秒逐币平仓冷却直接提交预测限价入场。
7. 反向动作本身可在每日停止门槛后执行，以便终止过时方向；新方向仍受每日次数和每日净亏损
   门槛限制。同向增加风险的动作在每日门槛触发后直接阻断。
8. V4.7 证据新增信号分发、预测加仓限价、加仓跳过、加仓并重设保护、反向请求和反向提交原因。

## 后果

V4.7 实现的是“最新已确认信号优先”，不是任意瞬时盘口变化优先。低质量、活跃度不足或尚未达到
连续轮次的方向不会接管仓位。同向加仓可能因剩余整仓风险、费用后目标、预测价格未成交或交易所
最小数量而被拒绝。反向新方向仍可能因预测限价未成交而保持空仓。这些限制是 Testnet 风险与
执行一致性边界，不构成胜率或收益保证。
