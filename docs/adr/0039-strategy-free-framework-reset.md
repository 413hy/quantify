# ADR 0039：重置为无策略交易框架

- 状态：已接受
- 日期：2026-07-16

## 背景

所有者计划基于现有工程基础重新开发一个新项目，并明确要求先删除当前交易策略及非必要内容。
继续保留 V4/V5 campaign、策略回放、策略仪表盘和数十份实验记录，会让新项目误把旧参数、旧结果
或旧服务当作有效默认值。

## 决策

1. 在账户无活动仓位和待入场请求时停止并禁用自动 campaign 与策略 Telegram 仪表盘。
2. 删除当前 PA/宽度/预测策略、策略持仓管理、策略专用 Testnet 执行器及其单元/回放测试。
3. 删除策略 campaign/dashboard systemd 单元、切换脚本、回测/复盘/结果汇总脚本。
4. 删除 V4/V5 策略 ADR、审计报告和仓库内策略证据；Git 历史仍保留此前提交，运行时历史证据不在
   本次源代码删除范围内。
5. 保留行情、特征计算、成本、风险、通用执行、原生保护、Testnet 安全探针、只读用户数据流、
   通知库、数据库和部署验证。
6. `ai_quant.strategy` 保留为空扩展点；框架不得自行选择币种、方向或入场时间。
7. 生产环境继续 `RISK_LOCKED`，保留服务均不具备自动下单权。

后续边界修正见 ADR 0040：旧策略 campaign 仍删除，但通用、无策略的自动交易引擎作为框架能力
恢复；这不改变当前运行服务无下单权的事实。

## 结果

新项目必须重新定义版本化决策契约、策略测试和独立部署单元，不能从旧 Git 历史静默恢复策略。
