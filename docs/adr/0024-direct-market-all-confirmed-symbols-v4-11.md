# ADR 0024：五币确认信号全部直接市价执行 V4.11

- 状态：Accepted
- 日期：2026-07-15
- 范围：Binance USDⓈ-M Futures Testnet；不解锁生产交易

## 背景

V4.9 在一轮同步下跌中生成 20 个计划，3 个确认信号均因 Testnet GTX 零成交而未建立仓位。
V4.10 使用 3 秒 Maker 后市价兜底，第一张自然确认的 XRP 做空信号通过市价成功成交 64.3 XRP，
并确认原生止盈止损均为 `NEW`。成交价相对信号参考价约差 0.9 bps，远小于此前等待限价造成的
整段行情遗漏。

账户所有者随后明确要求整个交易系统不再使用开仓限价；固定五币池中，任何币只要完成策略确认
就直接交易，不再保留每轮最高三个候选的限制。

## 决策

1. 删除活动执行路径中的 `LIMIT + GTX` 入口及其 Maker 等待实现。新仓和同向加仓只提交
   `MARKET`；反向信号仍先市价平旧方向，再市价开已确认的新方向。
2. 市价执行不降低信号标准。计划仍须通过方向 PA、实时主动成交、盘口或 microprice、点差、
   市场宽度、本币动量、成交活跃度、连续确认、最低质量分和预测信息量门槛。
3. 固定交易池仍为 BTCUSDT、ETHUSDT、BNBUSDT、SOLUSDT、XRPUSDT。每轮候选上限从 3 提升
   为 5；五个币同时独立确认时均可提交，最多同时持有五个不同币。
4. 五是容量上限，不是仓位目标。没有计划、活跃度不足、微观结构冲突或本币动量过热时允许空仓，
   不为补满仓位而交易。
5. 市价成交后必须使用交易所实际成交均价重新计算保护和费用后结果。预计净目标低于 0.10 USDT
   或预计净止损超过 1.00 USDT 时 fail-closed；成功时立即提交原生 `STOP_MARKET` 和
   `TAKE_PROFIT_MARKET`。fail-closed 平仓也必须形成完整结果并计入手续费和净盈亏。
6. 活动仓位只接收新一轮完整确认信号。同向信号在整仓风险和费用空间允许时市价加仓并重挂整仓
   保护；反向信号以最新确认方向接管。父仓已经退出时禁止同向市价加仓，避免意外重新开仓。
7. 快速联动入口与趋势入口使用一致的微观结构冲突否决：盘口或 microprice 任一项明显反向时
   不开仓，不能以另一项轻微同向掩盖强烈冲突。

## 后果

V4.11 不会再因 Testnet Maker 流动性造成确认信号零成交，但所有开仓都会承担 taker 手续费和
即时滑点。约 1 USDT 保证金、交易所最高初始杠杆和成交后复核限制单笔实验规模，不能保证盈利。
必须使用 V4.11 自然平仓样本分别审核各币的费用后胜率、Profit Factor、目标/止损平均值和滑点。
